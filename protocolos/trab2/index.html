Trabalho 2: Descoberta da rede via mensagens ICMP usando WebServices

Introdução

O trabalho desenvolvido consiste em uma aplicação que faz a determinação dos computadores ativos de uma rede local remotamente através de um webservice instalado na rede. O webservice executa o programa ping do trabalho anterior e manda a resposta para o computador requerente (cliente), que a exibe.

Código Fonte

O trabalho foi desenvolvido em PHP Ele pode ser obtido clicando aqui.

Funcionamento da Aplicação

O cliente requisita ao servidor a execução do programa ping (feita no trabalho 1), o qual, por sua vez, envia a resposta do ping pré-formatada para o cliente, que à exibe para ao usuário.

Projeto da Implementação

As principais funções da aplicação trab2_server.php são:

ping. Executa o ping broadcast.

$servidor->register('ping')Registra o ping no servidor, ou seja, deixa essa função visível para ser acessada remotamente pelo cliente.


A principal função da aplicação trab2_client.php é:

$soapclient->call('ping'). Colocada no cliente (trab2_client.php) serve para chamar a função ping que está no servidor (trab2_server.php).

Dificuldades Encontradas

Não foram encontradas grandes dificuldades no desenvolvimento do trabalho. Entretanto, tivemos alguns problemas para depurar o código; muitas vezes erros de sintaxe não eram reportados pelo php (como de fato deve acontecer) e isso dificultava a determinação de onde exatamente estava o problema.

Conclusão

A técnica utilizada neste trabalho é uma forma bastante eficiente de fazer a descoberta dos computadores que estão em uma rede local sem a necessidade de estar fisicamente no local (desde que esteja previamente instalado o suporte - apache, php, nusoap...).. Sua implementação é relativamente simples e a execução é rápida.
